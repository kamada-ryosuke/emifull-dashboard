# 給付費明細書 抽出ツール (PoC) セットアップ手順

このツールは、**障害児通所給付費・入所給付費等明細書（様式第二）のPDF**から
6項目を自動抽出してExcelに出力する検証用スクリプトです。

抽出項目：年月／事業所名／受給者証番号／障害児氏名／決定利用者負担額／請求額給付費

---

## 全体の流れ

```
[手順1] Python をインストール
   ↓
[手順2] Poppler をインストール
   ↓
[手順3] スクリプトの設定を環境に合わせる
   ↓
[手順4] 必要ライブラリをインストール
   ↓
[手順5] スクリプト実行
   ↓
[手順6] 抽出結果.xlsx を確認
```

所要時間の目安：**初回セットアップ30分＋初回実行5〜10分**（次回からは実行のみ）

---

## 手順1：Python のインストール

### 1-1. ダウンロード

ブラウザで以下を開きます：

```
https://www.python.org/downloads/
```

「**Download Python 3.11.x**」（または 3.12.x）の黄色いボタンを押してインストーラを保存。
※ 3.13 系はライブラリ未対応の可能性があるため **3.11 または 3.12 推奨**。

### 1-2. インストール

ダウンロードした `python-3.11.x-amd64.exe` を**ダブルクリック**。

**重要：インストーラ最初の画面で必ず以下にチェックを入れる**

- [x] **Add python.exe to PATH** ← 必ずチェック
- [x] Use admin privileges when installing py.exe

→ そのまま「**Install Now**」をクリック。

完了画面で「Disable path length limit」が出たらクリックしておく（推奨）。

### 1-3. 動作確認

スタートメニューで「**コマンド プロンプト**」を起動し、以下を入力：

```cmd
python --version
```

`Python 3.11.x` のように表示されればOK。

エラー（`'python' は内部コマンドまたは外部コマンド〜』）の場合：
- インストール時に「Add python.exe to PATH」にチェックを入れ忘れています
- アプリと機能から Python をアンインストールし、再度インストールしてください

---

## 手順2：Poppler のインストール

PDF をページ画像に変換するために必要です。

### 2-1. ダウンロード

ブラウザで以下を開きます：

```
https://github.com/oschwartz10612/poppler-windows/releases
```

最新リリースの **Assets** から `Release-XX.XX.X-0.zip`（数字部分は最新版）をダウンロード。

### 2-2. 展開

1. ダウンロードしたzipを **Cドライブ直下に展開**
2. 展開後、フォルダ名を `poppler` にリネーム（短い方が扱いやすい）

最終的に以下のような構造になればOK：

```
C:\poppler\
  └─ Library\
      └─ bin\
          ├─ pdftoppm.exe
          ├─ pdfinfo.exe
          └─ ...
```

### 2-3. パスを確認

`C:\poppler\Library\bin` フォルダを開き、`pdftoppm.exe` が**存在することを確認**してください。
（バージョンによりサブフォルダ名が違うことがあります。`pdftoppm.exe` がある階層のフルパスをメモしておく）

---

## 手順3：スクリプトの設定を環境に合わせる

`extract_pdf.py` をメモ帳または VSCode で開き、ファイル冒頭の **CONFIG セクション**を確認します：

```python
# ============================================================
# CONFIG  ここを環境に合わせて修正してください
# ============================================================
PDF_PATH = r"C:\Users\user01\OneDrive\デスクトップ\給付費明細書.pdf"
POPPLER_PATH = r"C:\poppler\Library\bin"   # ← 手順2-3 で確認したパス
WORK_DIR = Path(r"C:\Users\user01\OneDrive\デスクトップ\売上入金管理ツール")
```

修正箇所：
- **POPPLER_PATH** … 手順2-3で確認した `pdftoppm.exe` があるフォルダのフルパス
- **PDF_PATH** … 抽出したいPDFのフルパス（変える場合）

修正後、保存して閉じる。

## 手順4：必要ライブラリのインストール

### 4-1. コマンドプロンプトで作業フォルダに移動

スタートメニューで「**コマンド プロンプト**」を起動し、以下を入力：

```cmd
cd /d "C:\Users\user01\OneDrive\デスクトップ\売上入金管理ツール"
```

`売上入金管理ツール>` のように表示されればOK。

### 4-2. pip をアップデート

```cmd
python -m pip install --upgrade pip
```

### 4-3. 依存ライブラリをインストール

```cmd
pip install -r requirements.txt
```

**所要時間：5〜10分**（PaddlePaddleが大きいため）

途中で警告が出ても、最後に `Successfully installed ...` と表示されればOK。

エラーが出た場合は、エラーメッセージをコピーしてClaudeに共有してください。

## 手順5：スクリプト実行

同じコマンドプロンプトで：

```cmd
python extract_pdf.py
```

### 初回実行時の挙動

1. **OCRモデルの自動ダウンロード**（約 200〜500 MB、数分かかります）
   - 「Downloading...」のような表示が出ます
   - 一度ダウンロードすれば次回以降は不要
2. PDFをページ画像化
3. 各ページを順次OCR（**1ページ 8〜15秒**、25ページなら 4〜7分）
4. 抽出結果を Excel に書き出し

## 手順6：抽出結果の確認

作業フォルダの `抽出結果.xlsx` を開きます。

| ページ | 年月 | 事業所名 | 受給者証番号 | 障害児氏名 | 決定利用者負担額 | 請求額給付費 | _信頼度_… |
|---|---|---|---|---|---|---|---|
| 1 | 2026-01 | SORATOてんり | 0600018543 | 河*正* | 4600 | 104234 | … |

### 確認ポイント

- すべての行の値が正しいか
- `_信頼度_…` 列が 0.9 以上か（低い行は OCR が自信なし＝要確認）
- 抜け（空欄）や明らかな誤読（金額の桁違い、氏名が変）がないか

### 精度が悪い場合

`debug\images\page_001.png` を開けば、OCRが見た画像が確認できます。
`debug\ocr_raw\page_001.json` には、検出された全テキストとその位置情報が入っています。

→ これらを確認した上で、Claude にどの項目がどう間違っているかを教えてください。
   抽出ロジックを調整します。

## トラブルシューティング

### `'python' は内部コマンドまたは外部コマンド…として認識されていません`
→ 手順1のPATH追加にチェックを入れていません。Pythonを再インストールしてください。

### `[ERROR] Popplerが見つかりません`
→ `POPPLER_PATH` の設定が間違っています。手順2-3 で確認したパスを再設定してください。

### `[ERROR] PDFが見つかりません`
→ `PDF_PATH` を修正してください。

### `Memory error` / OCR中にフリーズ
→ メモリ不足の可能性。他のアプリを閉じてから再実行してください。
   特にChrome等を閉じるとメモリが空きます。

### OCRがすごく遅い
→ 想定どおりです。1ページ8〜15秒、25ページで4〜7分かかります。
   実際の運用（120ページ）では15〜30分が目安です。
