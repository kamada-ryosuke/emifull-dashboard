================================================================
  デビットカード レシート自動転記システム ─ 操作マニュアル
================================================================

【概要】
スマホで撮ったレシートをDropboxの該当施設フォルダにアップすると、
ClaudeのAIが自動で日付・店舗・金額を読み取り、経理管理番号付きで
出力Excelに月次シートとして転記します。手書き施設名は不要(フォルダ名から判別)。

【運用ルール】
----------------------------------------------------------------
■ 単一施設の場合
  ・ レシートを撮影 → 該当施設フォルダにアップロード
  ・ 手書きで施設名を書く必要なし(フォルダで判別)

■ 振替ルール(06　大西キッチン)
  ・ 大西キッチンで使ったものは「06　大西キッチン」フォルダにアップ
  ・ Excelには「02　のじぎく稲美」として記録 + 摘要に「（大西キッチン用）」追記される
  ・ 他にも振替が必要な施設があれば config\facilities.json の folder_overrides に追記

■ 按分(複数施設で割り勘)
  ・ レシートの余白に施設名を複数書く + 「○按分」と書く
    例: 「SORATOてんり、UMIEてんり、カラダキッズてんり 3按分」
  ・ 表記ゆれOK: 「按分」「あんぶん」「按文」「案分」「均等割」「等分」全て認識
  ・ 区切りも自由: 「、」「／」「・」「と」「&」「+」全部OK
  ・ 代表で1施設フォルダ(任意)にアップ → 自動でN施設に等分転記
  ・ 端数(円未満)は config\revenue_ranking.json の上位施設に集中

■ 施設名の手書き表記
  ・ 平仮名/カタカナ/ローマ字/漢字どれでも認識(ファジー一致)
  ・ 例: 「UMIEてんり」「ウミエてんり」「うみえてんり」「UMIE天理」「umie tenri」全部OK

【日々の使い方】
----------------------------------------------------------------
■ 自動転記モード(常時稼働 / おすすめ)
  1. C:\売上入金管理ツール\scripts\start_watcher.bat をダブルクリック
  2. 「監視中...」と表示されたまま放置
  3. スマホやPCからDropboxの該当施設フォルダに画像を置くだけで自動転記
  4. 終了は Ctrl+C

■ 一括処理モード
  1. 該当施設フォルダにレシートをまとめて入れる
  2. C:\売上入金管理ツール\scripts\process_all.bat をダブルクリック

【出力先】(Dropbox直下)
----------------------------------------------------------------
  \株式会社ＥＭＩＦＵＬＬ Dropbox\医療法人社団EMIFULL\障がい事業部\00障がい事業部\
    ├── 医療法人EMIFULL_デビット.xlsx
    └── NPO法人EMIFULL_デビット.xlsx

各Excelには以下の12列が出力されます(既存.xlsmと同じA-K列 + 新設L列):
  A:日付  B:借方勘定科目  C:借方税区分  D:借方品目  E:借方部門  F:借方金額
  G:貸方勘定科目  H:貸方品目  I:貸方部門  J:貸方金額
  K:摘要  ← 店舗名+用途+購入商品(高額順、5件以上、略号は補完)
  L:検証  ← 「✓ 3重チェック OK」または「⚠ 要確認」(要確認は黄色で強調)

----------------------------------------------------------------
【精度を支える3つの仕組み】
----------------------------------------------------------------
■ 3重チェック (各レシートに自動適用)
  Level 1: AI画像読み取り(Claude Opus)
  Level 2: 内部整合性チェック(金額・日付・施設・税区分の論理検証)
  Level 3: 別モデル(Claude Sonnet)で合計・日付・店舗を独立に再読取して照合
  → どれかで違和感があれば L列が「⚠ 要確認」になり該当行が黄色で表示。

■ 内容ベースの2重転記防止
  ・ファイルハッシュ(完全一致): 同じファイルを2回入れたら即スキップ
  ・内容シグネチャ(日付+施設+金額): 別ファイル名でも内容が同じなら検出
  → 検出された2回目の画像は _要確認\重複_2重アップ\<施設>\ に保管。
    Excelには重複転記されない。

■ 商品リストを摘要に自動追記
  ・各レシートから購入商品を「金額の高い順」に最低5件抽出
  ・略称・誤読は推測補完(例: TVサクチョコチクッキ → TVサクサクチョコクッキー)
  ・形式: 「店舗 用途 / 商品A¥XXX, 商品B¥XXX, ...」

各Excelには月別シート(例: 2605【デビット】= 26年5月分)が自動作成され、
日付・施設・摘要順にソートされます。月跨ぎは自動。

----------------------------------------------------------------
【マスター調整方法】
----------------------------------------------------------------
■ 新施設を追加 / 既存施設の別名を増やす
  config\facilities.json を編集
  該当法人の facilities 配列に追加 or aliases に追記

■ 新店舗・勘定マッピングを追加
  config\account_mapping.json の rules 配列に追加

■ 振替ルールを追加(○○フォルダのレシートを△△施設として記録)
  config\facilities.json の folder_overrides に追記

■ 按分の端数優先順位(売上ランキング)を更新
  config\revenue_ranking.json を月次で編集
  ※ 直近月の売上が高い順に並べ替えてください

----------------------------------------------------------------
【トラブルシューティング】
----------------------------------------------------------------
■ _要確認 にレシートが入った
  → 撮り直す or 手動でExcelに追記
  → logs\processing.log にエラー理由が出ます

■ 施設コードが「(要確認)」で出る
  → facilities.json の aliases に書き方バリエーションを追加

■ 同じ画像を再処理したい
  → logs\processed_files.json から該当ハッシュ行を削除

■ Excelが開きっぱなしだと書き込みエラー
  → Excel を閉じてから自動転記する

■ APIキーが無効
  → ANTHROPIC_API_KEY を再設定 → コマンドプロンプトを開き直す
